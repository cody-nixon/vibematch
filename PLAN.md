# VibeMatch — Technical Plan

## Product Identity

**Name:** VibeMatch  
**Tagline:** "Find your next favorite game in 60 seconds."  
**Domain:** vibematch.gg (or vibematch.io)

**One-liner:** VibeMatch learns how gaming *feels* to you and surfaces indie games you'd love — even if you've never heard of them.

---

## User Stories (Only What Matters)

### Core Flow
1. **As a new user**, I answer 6 vibe questions about how I like to feel when I play games, so VibeMatch understands my taste before I even log in.
2. **As a user**, I can search and select games I've loved in the past to calibrate my taste fingerprint.
3. **As a user**, I receive 10 personalized game recommendations immediately, each with a one-sentence "why you'd love this" explanation.
4. **As a user**, I can swipe/rate recommendations (Love / Pass / Already Played), and my fingerprint updates in real time.
5. **As a user**, I filter recommendations by: available on Steam, free-to-play, time-to-beat (under 5h / 5-15h / 15h+), price range, multiplayer/solo.
6. **As a user**, I receive a weekly "New Drops" email with 3 new releases that match my vibe profile.
7. **As a user**, I can share my VibeMatch profile as a public link ("Here's my gaming vibe — what matches?").

### Discovery Enhancement
8. **As a user**, I can type a "vibe sentence" — "I want something lonely and beautiful with no combat" — and get instant matching results.
9. **As a user**, I can compare my taste profile with a friend's and see "games you'd both love."

### Out of Scope (v1)
- Social features / following other users
- Game library sync (Steam API import of owned games — v2)
- Mobile app
- Community reviews
- Developer portal (listing your own game — v2)

## Tech Stack

| Layer | Choice | Rationale |
|-------|--------|-----------|
| Frontend | Next.js 14 (App Router) + TailwindCSS + shadcn/ui | SSR for SEO (game pages must be indexable), fast dev velocity |
| Backend | Next.js API routes (Node.js) | Colocated with frontend, simple deployment |
| Database | PostgreSQL (Supabase) | Relational for users/games/ratings, pgvector for similarity search |
| Game embeddings | pgvector + OpenAI text-embedding-3-small | Semantic similarity between game descriptions and vibe fingerprints |
| Game data | IGDB API (primary) + Steam API (secondary) | IGDB: rich metadata, screenshots, tags. Steam: pricing, reviews. Both free. |
| Auth | Supabase Auth (email/magic link + Steam OAuth) | Steam OAuth is perfect for this use case — players already have Steam |
| Email | Resend | Simple, cheap, great DX |
| Deployment | Vercel | Zero-config Next.js deployment |
| Background jobs | Vercel Cron (or Trigger.dev free tier) | Weekly email digests, nightly game library updates |

**Why pgvector over Pinecone/Weaviate:** Keeping embeddings in Postgres avoids a third service. For the game library size we're dealing with (50K-200K games), pgvector is plenty fast. Simpler architecture.

## Architecture Overview

```
User → Browser (Next.js)
         ↓
    Vibe Quiz / Game Selection
         ↓
    API: POST /api/taste/calibrate
         ↓
    Generates Taste Embedding (weighted avg of game embeddings + quiz vector)
         ↓
    pgvector: cosine similarity search across game_embeddings table
         ↓
    Returns ranked games with explanation text (generated by LLM)
         ↓
    User rates → PATCH /api/taste/rate
         ↓
    Re-weighting of taste vector (positive ratings amplify, negative dampen)
         ↓
    Weekly Cron: new games ingested → matched against all user taste vectors → email sent
```

### Game Data Pipeline
```
IGDB API (nightly sync) 
    → normalize game metadata (title, description, genres, themes, keywords, screenshots)
    → generate embedding via OpenAI text-embedding-3-small
    → store in games table with embedding column (pgvector)
    → update Steam price/rating data
```

## Data Model

```sql
-- Core tables

CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email TEXT UNIQUE NOT NULL,
    steam_id TEXT UNIQUE,
    display_name TEXT,
    avatar_url TEXT,
    taste_embedding VECTOR(1536), -- user's current taste fingerprint
    taste_calibrated_at TIMESTAMPTZ,
    quiz_answers JSONB, -- raw quiz responses for recalibration
    email_digest_enabled BOOLEAN DEFAULT true,
    created_at TIMESTAMPTZ DEFAULT now(),
    updated_at TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE games (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    igdb_id INTEGER UNIQUE,
    steam_appid INTEGER UNIQUE,
    title TEXT NOT NULL,
    slug TEXT UNIQUE NOT NULL, -- for URL: /game/hollow-knight
    description TEXT,
    short_description TEXT, -- 1-2 sentence summary
    cover_url TEXT,
    screenshots JSONB, -- array of URLs
    genres TEXT[],
    themes TEXT[],
    keywords TEXT[],
    developer TEXT,
    publisher TEXT,
    release_date DATE,
    steam_rating DECIMAL(4,1), -- e.g. 95.2
    steam_review_count INTEGER,
    steam_price_usd DECIMAL(8,2),
    steam_url TEXT,
    avg_time_to_beat_hours DECIMAL(6,1), -- from HowLongToBeat
    platforms TEXT[], -- ['PC', 'Switch', 'PS5', etc.]
    is_free BOOLEAN DEFAULT false,
    is_multiplayer BOOLEAN DEFAULT false,
    embedding VECTOR(1536), -- semantic embedding of description+keywords
    last_synced_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT now()
);

-- Index for similarity search
CREATE INDEX games_embedding_idx ON games USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);
CREATE INDEX users_taste_embedding_idx ON users USING ivfflat (taste_embedding vector_cosine_ops) WITH (lists = 50);

CREATE TABLE user_ratings (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    game_id UUID REFERENCES games(id) ON DELETE CASCADE,
    rating TEXT NOT NULL CHECK (rating IN ('love', 'pass', 'played', 'wishlisted')),
    source TEXT DEFAULT 'recommendation', -- 'recommendation' | 'search' | 'calibration'
    created_at TIMESTAMPTZ DEFAULT now(),
    UNIQUE(user_id, game_id)
);

CREATE TABLE recommendation_history (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    game_id UUID REFERENCES games(id),
    similarity_score DECIMAL(6,4),
    why_text TEXT, -- "Because you loved Hollow Knight's atmosphere and challenge"
    shown_at TIMESTAMPTZ DEFAULT now(),
    clicked BOOLEAN DEFAULT false,
    steam_click BOOLEAN DEFAULT false
);

CREATE TABLE digest_sends (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    game_ids UUID[],
    sent_at TIMESTAMPTZ DEFAULT now()
);
```

## API Design

### Taste Calibration
```
POST /api/taste/calibrate
Body: {
  quiz_answers: {
    session_length: "20min" | "1hr" | "3hr+",
    difficulty_mood: "relaxed" | "challenged" | "mastery",
    world_style: "cozy" | "dark" | "vibrant" | "minimalist",
    social: "solo" | "coop",
    narrative_weight: "story_driven" | "emergent" | "no_story",
    reward_type: "progress" | "exploration" | "competition" | "creation"
  },
  seed_game_igdb_ids: [123, 456, 789] // games they already love
}
Response: {
  taste_profile_id: "uuid",
  recommendations: [GameRecommendation],
  taste_summary: "Your vibe: cozy exploration with emotional narrative, best in 1-hour sessions"
}
```

```
PATCH /api/taste/rate
Body: { game_id: "uuid", rating: "love" | "pass" | "played" }
Response: { updated_recommendations: [GameRecommendation] }
```

### Game Search
```
GET /api/games/search?q=hollow+knight
Response: { games: [GamePreview] }

POST /api/games/vibe-search
Body: { query: "I want something lonely and beautiful with no combat" }
Response: { games: [GameRecommendation], interpreted_as: "calm atmospheric exploration" }
```

### Recommendations
```
GET /api/recommendations?limit=10&offset=0&platform=PC&max_price=20&max_hours=10
Response: { games: [GameRecommendation], next_page_token: "..." }
```

### Profile Sharing
```
GET /api/profile/:username/public
Response: { taste_summary, top_loved_games, public_recommendations }

GET /api/profile/compare?user_a=:id&user_b=:id
Response: { shared_loves, recommended_for_both }
```

## The Taste Embedding Algorithm

### Initial Calibration
1. Map quiz answers to a semantic query string:
   - `"relaxed cozy exploration story-driven solo 1-hour sessions"` 
2. Embed this query string using OpenAI text-embedding-3-small
3. Embed each seed game's description
4. Compute weighted average: quiz embedding (30% weight) + mean of seed game embeddings (70% weight)
5. Store as user's `taste_embedding`

### Learning from Ratings
- `love`: Pull taste vector toward this game's embedding (step size: 0.05)
- `pass`: Push taste vector away from this game's embedding (step size: 0.02)
- `played`: Neutral (exclude from future recommendations, no vector change)
- Normalize vector after each update

### Why-Text Generation
For each recommendation, generate explanation:
```
prompt: "In one sentence, explain why a gamer who loves [loved_games] and prefers [taste_summary] would enjoy [game_title]: [game_description]"
```
Cache these explanations per (user_taste_hash, game_id) pair to avoid repeated LLM calls.

## Auth Flow

1. **Option A: Magic Link Email** (frictionless, no password)
   - Enter email → receive link → auto-logged in → taste carries over
2. **Option B: Steam OAuth** (ideal for gamers)
   - "Sign in with Steam" → auto-imports Steam display name + avatar
   - Future: can read public library to see what you own (v2)

**Guest mode:** Taste quiz + recommendations work without account. Account required for: rating history, email digest, sharing.

## UI Flow (Step by Step)

### 1. Landing Page
- Hero: "Find your next favorite indie game in 60 seconds"
- CTA: "Find My Match" button (no signup required)
- Social proof: "3,847 indie gems discovered this week"

### 2. Vibe Quiz (6 questions, one at a time, swipeable cards)
- Q1: "How long do your gaming sessions usually run?"
- Q2: "When you think of your favorite game moments, they felt..."
- Q3: "The world of your perfect game looks like..."
- Q4: "You prefer playing..."
- Q5: "How important is story to you?"
- Q6: "What keeps you playing?"

### 3. Seed Games Search
- "Name 1-3 games you love (any game, any platform)"
- Autocomplete search against IGDB database
- Optional step (skip works fine, quiz alone is enough)

### 4. Results — Your Match Stack
- Card-based UI, 10 games shown
- Each card: cover art, title, "Why you'd love this" in one sentence, rating (87% positive), price, time-to-beat
- Swipe left (Pass) / right (Love) / up (Add to wishlist)
- Steam affiliate links on "View on Steam" button

### 5. Refined Results (after 3+ ratings)
- Results visibly improve as ratings come in
- "Your taste is getting clearer — keep going!"

### 6. Account Save Prompt (after 5 ratings)
- "Save your vibe profile to get weekly new game alerts"
- Email or Steam sign-in

### 7. Public Profile Page
- vibematch.gg/@username
- Shows: Taste Summary, Top 5 loved games, Current Top Matches
- Shareable link

## Background Jobs / Cron

```
DAILY (2 AM UTC):
- Sync new games from IGDB (last 24h releases)
- Generate embeddings for new games
- Update Steam ratings/prices for all games

WEEKLY (Monday 9 AM user timezone):
- For each user with email digest enabled:
  - Find new games added in last 7 days that match taste vector
  - Select top 3 with similarity > 0.75
  - Generate why-text for each
  - Send digest email via Resend

MONTHLY:
- Recalibrate taste vectors for users who haven't rated anything in 30 days
```
